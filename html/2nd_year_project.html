Hospital Waiting Room System

A real-time healthcare tracking system developed in Year 2

Project Overview

This system was created to monitor patient flow and waiting room activity inside a hospital environment. Patients arriving at reception are checked in by a receptionist, who may grant or deny access based on their medical issues. Once granted access, a digital tag is assigned to the patient containing metadata such as Tag ID, Location ID and timestamp.

Sensors detect these tags as patients move through hospital areas. The system updates patient logs, calculates waiting time and triggers status changes (Vacant, Busy or Crowded). After the patient is finished, the receptionist checks them out and updates the record database.

MQTT Messaging System

MQTT was used for real-time event updates as patients moved through the hospital. The system published live messages using mosquitto_pub and stored the received data into the SQL database, which was later displayed on the dashboard.
